# 个人待办增加搜索HTTP API接口 - API接口设计

## 1. 接口概述

本文档描述个人待办模块新增的搜索HTTP API接口，包括接口路径、请求参数、响应结果以及处理逻辑。

## 2. 接口基本信息

| 项目 | 内容 |
|------|------|
| 接口名称 | 搜索待办任务 |
| 接口路径 | `GET /api/todo/search` |
| 请求方式 | GET |
| 认证方式 | 需要登录认证 |
| 所属模块 | 个人待办模块 (Todo) |
| Controller | TodoListController |

## 3. 接口详细设计
### 3.1 搜索待办任务
#### 3.1.1 接口定义
**接口路径**: `GET /api/todo/search`
**请求方式**: GET
#### 3.1.2 请求参数
| 参数名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|------|--------|
| keyword | String | 是 | 搜索关键词，支持模糊匹配待办标题和待办ID | "项目文档" |
| pageNum | Integer | 否 | 页码，默认值为1 | 1 |
| pageSize | Integer | 否 | 每页数量，默认值为10，最大值为100 | 10 |

**请求示例**:

```
GET /api/todo/search?keyword=项目文档&pageNum=1&pageSize=10
```
**参数说明**:
- `keyword`: 搜索关键词，会对待办标题（content字段）和待办ID（id字段）进行模糊匹配
- `pageNum`: 页码，从1开始计数
- `pageSize`: 每页返回的记录数，建议不超过100条

#### 3.1.3 响应结果
**响应格式**: JSON

**响应示例**:

```json
{
  "code": "200",
  "message": "success",
  "data": {
    "total": 25,
    "list": [
      {
        "id": "123456",
        "title": "完成项目文档",
        "deadLine": "截止时间",
      }
    ],
    "pageNum": 1,
    "pageSize": 10
  }
}
```
