# 个人待办增加获取执行人下拉列表接口 - API接口设计

## 1. 接口概述

### 1.1 接口说明
本接口用于获取当前登录用户创建的待办任务的所有执行人列表，以供前端下拉选择使用。

### 1.2 接口分类
个人待办接口 (Todo)

## 2. 接口详情

### 2.1 获取执行人下拉列表

#### 2.1.1 接口基本信息
- **接口名称**: 获取执行人下拉列表
- **接口路径**: `GET /api/todo/user/list`
- **请求方式**: GET
- **是否需要登录**: 是
- **接口描述**: 根据当前登录用户，查询该用户创建的待办任务的所有执行人列表

#### 2.1.2 请求参数
无需请求参数，接口自动从登录信息获取当前用户身份。

#### 2.1.3 请求示例
```http
GET /api/todo/user/list HTTP/1.1
Host: localhost:8080
Content-Type: application/json
Authorization: Bearer {token}
```

#### 2.1.4 响应参数

| 字段名 | 类型 | 说明 |
|--------|------|------|
| code | String | 响应码，"0000"表示成功，其他表示失败 |
| message | String | 响应消息 |
| data | Array | 执行人列表 |

#### 2.1.5 响应示例

**成功响应**:
```json
{
  "code": "200",
  "message": "success",
  "data": [
      "1001",
      "1002",
      "1003"
  ]
}
```

**空数据响应**:
```json
{
  "code": "200",
  "message": "success",
  "data": []
}
```

**未登录响应**:
```json
{
  "code": "401",
  "message": "未登录",
  "data": null
}
```

**系统异常响应**:
```json
{
  "code": "500",
  "message": "系统错误",
  "data": null
}
```
